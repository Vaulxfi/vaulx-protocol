5. EDSON — Lead Developer / Solana Protocol Engineer
Role: Smart contract development, cNFT minting system, vault architecture, on-chain loan lifecycle, frontend dApp integration.

Background:
Edson is an experienced developer carrying the full technical execution of Vaulx's on-chain infrastructure. The technical stack required for Vaulx is specific and demanding:

Anchor Framework on Solana: The primary Rust-based framework for writing Solana smart contracts. Anchor abstracts Solana's account model complexity and provides structured constraint validation — essential for a lending protocol where logic errors can result in fund loss.

Metaplex Bubblegum: The compressed NFT standard on Solana used to mint cNFTs at near-zero cost (less than USD 0.001 per transaction). Vaulx's custody proof layer mints a cNFT at asset intake — Edson's familiarity with Bubblegum is non-negotiable for this function.

Vault and pool architecture: Crowdlending vaults require atomic transaction logic for deposits, withdrawals, interest accrual, and pro-rata liquidation distribution — all requiring battle-tested Solana program design.

IPFS / Arweave integration: Appraisal data, asset photography, and condition records are stored off-chain and linked via the cNFT metadata URI. Edson's experience integrating permanent decentralized storage is required for the custody chain to be legally auditable.

Oracle integration: Depeg monitoring for BRL stablecoins and real-time LTV enforcement require price oracle connections — Pyth Network or Switchboard on Solana.

Why this matters for Vaulx:

A DeFi protocol with physical collateral has one additional attack surface that pure on-chain protocols do not: the interface between the physical world and the blockchain. The cNFT mint event is the moment where a physical reality is encoded as an on-chain fact. If this encoding process is compromised — if an asset is minted without verified custody, or if a custody release is triggered without valid loan repayment — the entire protocol collapses.

Edson's developer experience means the team does not depend on outsourced development for its core security-critical code. This is important for two reasons. First, it reduces time-to-MVP — the hackathon submission requires a working devnet deployment, not a mockup. Second, it reduces long-term technical risk. RWA protocols have been exploited through smart contract vulnerabilities at significant scale. Having a dedicated in-house developer who owns the codebase end-to-end is the correct security posture for a protocol handling physical assets.

Investor signal: A working devnet demo at submission time is a Colosseum requirement that separates serious teams from idea-stage applicants. Edson's presence means Vaulx submits with code, not slides.
